Tabela – Variáveis selecionadas

Selecione as variáveis que você deseja que sejam mostradas na tabela de dados. Você pode fazer isso (1) pressionando sobre o nome das variáveis no menu suspenso ou (2) digitando o nome das variáveis na caixa de seleção. Para remover variáveis da seleção, basta posicionar o cursor do mouse junto do nome da mesma e pressionar as teclas Delete ou Backspace.

Mapa – Raio de agrupamento

Defina o modo de apresentação espacial das observações do solo. Você pode utilizar a barra de rolagem para selecionar um valor entre 0 e 100 pixels. Quanto maior o valor do raio, mais observações serão agregadas sob um mesmo agrupamento e, assim, um número menor de agrupamentos será formado. Defina o raio como zero para reduzir ao máximo o agrupamento de observações. Note que a alteração do raio de agrupamento força o retorno do mapa à sua extensão original.

Tabela

Visão geral

As variáveis selecionadas no painel lateral são usadas para construir a tabela de dados. Três variáveis de identificação são mostradas por padrão: dataset_id, o código de identificação do conjunto de dados, observacao_id, o código de identificação das observações/perfis do solo, e camada_id, o código de identificação das camadas/amostras do solo. A disposição inicial dos dados é feita seguindo a ordem crescente do dataset_id, com cinco linhas, onde cada linha representa uma camada/amostra do solo. O número de linhas mostradas pode ser alterada no topo da tabela junto à opção "resultados por página". Na parte inferior da tabela está indicado o número total de registros. Para navegar ao longo dos registros, basta utilizar os controles localizados no canto inferior direito abaixo da tabela. Caso muitas variáveis sejam selecionadas, a tabela ficará maior do que o espaço disponível em tela. Nesse caso, será preciso usar a barra de rolagem horizontal do navegador para visualizar todas as colunas da tabela.

Filtros de coluna

A principal funcionalidade de busca de dados processados são os filtros de coluna. Depois de adicionaras variáveis de interesse, você pode estabelecer critérios para filtrar conjuntos de dados, observações/perfis e camadas/amostras de solo. Esses critérios são inseridos na caixa localizada imediatamente abaixo do nome de cada coluna da tabela de dados. Por exemplo, se você tiver interesse no conjunto de dados ctb0003, basta inserir esse código na caixa de filtragem da coluna dataset_id. Com isso, a tabela de dados mostrará apenas os dados desse conjunto de dados.

O motor de filtragem possui duas características importantes. A primeira delas é a insensibilidade à caixa das letras, ou seja, os filtros são insensíveis a maiúsculas e minúsculas. Isso significa que você terá o mesmo resultado de filtragem se usar Latossolo, LATOSSOLO ou latossolo como critério na coluna com dados de classificação taxonômica (taxon_sibcs). Entretanto, você deve tomar muito cuidado com acentos e outros caracteres especiais, já que os filtros são incapazes de substituir caracteres acentuados por homólogos não-acentuados. Isso significa que, por exemplo, distrófico e distrofico não retornarão os mesmos resultados.

A segunda característica importante do motor de filtragem é a habilidade de trabalhar com expressões regulares (regex, do inglês regular expression). Com as expressões regulares é possível especificar critérios utilizando operadores para identificar cadeias de caracteres específicas. Vejamos alguns exemplos:

  • Alternativas. O operador | é o operador de alternância. Ele testa a ocorrência de cada uma das cadeias de caracteres especificadas. Assim, o critério de busca distrófico|distrofico significa "retorne todas as ocorrências em que há distrófico OU distrofico". Com esse operador também é possível filtrar os dados de dois ou mais estados, por exemplo, RS|SC|PR – ou de outras variáveis.
  • Âncoras. Os operadores ^ e $ são operadores de âncora. Eles testam o ocorrência de uma cadeia de caracteres em uma determinada posição do texto, seja no início (^) ou no fim ($). Por exemplo, filtrando dados por município (municipio_id), ^João retorna os dados dos municípios cujo nome inicia por "João". Já João$ retorna os dados daqueles cujo nome termina por "João".

Mais informações sobre expressões regulares e operadores estão disponíveis no artigo da Wikipédia sobre o tópico.

Opções de descarregamento

As seguintes opções de descarregamento de (conjuntos de) dados estão disponíveis:

  • Amostra(s) filtrada(s). Dados processados das amostras (camada_id) selecionadas usando os filtros de coluna na tabela de dados. O arquivo descarregado consiste num arquivo de texto plano de extensão TXT, em que as colunas de dados são separadas por uma tabulação. O tipo de codificação de caracteres usado é o UTF-8.
  • Local(is) filtrado(s). Dados processados dos locais (observacao_id) selecionados usando os filtros de coluna na tabela de dados.
  • Conjunto(s) de dados filtrado(s). Dados processados de todos os locais e amostras contidas nos conjuntos de dados (dataset_id) filtrados na tabela de dados.
  • Conjunto(s) de dados original(is). Dados originais de todos os locais e amostras contidas nos conjuntos de dados (dataset_id) filtrados na tabela de dados. O arquivo descarregado consiste num arquivo compactado no formato ZIP, em que os arquivos constituintes são arquivos no formato XLSX, um para cada conjunto de dados filtrado. Com os dados originais em mãos, você pode aplicar os processamentos de sua preferência.
  • Todos os conjuntos de dados. Dados originais de todos os conjuntos de dados (dataset_id) publicados no repositório. O arquivo descarregado consiste num arquivo compactado no formato ZIP, em que os arquivos constituintes (nos formatos CSV e XLSX) estão dispostos em diretórios, um para cada conjunto de dados existente. Com os dados originais em mãos, você pode aplicar os processamentos de sua preferência.
